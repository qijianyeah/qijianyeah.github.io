Spring 中 doCreateBean 方法详解

doCreateBean 是 Spring IoC 容器中创建 Bean 实例的核心方法，位于 AbstractAutowireCapableBeanFactory 类中。它实现了 Bean 实例化、属性填充和初始化的完整流程。

一、方法签名与总体流程

总体流程图：

二、详细执行步骤

1. 实例化 Bean 对象

关键点：

  • 使用 createBeanInstance 方法通过反射或工厂方法创建实例
  • 返回 BeanWrapper 对象包装原始 Bean
  • 处理构造器注入（如果有）

2. 应用 MergedBeanDefinitionPostProcessor

作用：

  • 处理合并后的 BeanDefinition
  • 典型应用：AutowiredAnnotationBeanPostProcessor 在此阶段查找 @Autowired 和 @Value 注解

3. 提前暴露引用（解决循环依赖）

三级缓存机制：

  1. singletonFactories（三级缓存）：存储单例工厂
  2. earlySingletonObjects（二级缓存）：存储早期引用
  3. singletonObjects（一级缓存）：存储完全初始化的 Bean

4. 属性填充（依赖注入）

populateBean 主要工作：

  • 处理 @Autowired 字段/方法注入
  • 处理 @Value 注解
  • 处理 XML 中定义的 <property> 元素
  • 应用 InstantiationAwareBeanPostProcessor

6. 注册 DisposableBean

处理销毁逻辑：

  • @PreDestroy 注解方法
  • DisposableBean 接口
  • 自定义 destroy-method

三、关键扩展点

1. InstantiationAwareBeanPostProcessor

2. MergedBeanDefinitionPostProcessor

典型实现：

  • AutowiredAnnotationBeanPostProcessor：处理自动注入注解
  • CommonAnnotationBeanPostProcessor：处理 @PostConstruct 等注解

六、常见问题与调试技巧

1. 调试技巧

  • 断点位置：
    • AbstractAutowireCapableBeanFactory.doCreateBean()
    • DefaultSingletonBeanRegistry.getSingleton()
    • AbstractBeanFactory.getBean()

2. 常见问题

问题1：循环依赖报错
解决：检查是否使用了构造器注入，考虑改为 setter 注入

问题2：@PostConstruct 方法未执行
解决：检查是否缺少 CommonAnnotationBeanPostProcessor

问题3：属性未注入
解决：检查 populateBean 阶段日志，确认属性是否被正确识别

doCreateBean 是 Spring IoC 容器最核心的方法之一，理解它的执行流程和扩展点对于深入掌握 Spring 框架至关重要。通过合理利用各种后处理器，可以实现强大的定制功能。
