Spring 容器生命周期详解

Spring 容器的生命周期是理解 Spring 框架运作机制的核心。下面我将详细展开每个阶段的运作原理和关键点：

1. Bean 定义加载阶段

核心过程：

  • 容器读取配置元数据（XML、Java 注解或代码）
  • 解析并转换为 BeanDefinition 对象
  • 注册到 BeanDefinitionRegistry

关键点：

  • 配置来源：

    • XML 配置文件（<bean> 标签）
    • Java 注解（@Component, @Bean 等）
    • 编程方式（BeanDefinitionBuilder）

  • 扩展点：

典型场景：

  • 动态注册 Bean（如根据数据库表生成 Repository）
  • 条件化 Bean 定义（Profile 的实现原理）

2. Bean 实例化阶段

核心过程：

  • 根据 BeanDefinition 创建 Bean 实例
  • 主要使用反射机制（Constructor.newInstance()）

关键点：

  • 实例化策略：

    • 默认使用简单实例化策略（SimpleInstantiationStrategy）
    • 需要方法重写的类使用 CGLIB 增强

  • 扩展点：

典型问题：

  • 循环依赖的解决（三级缓存机制）
  • 代理对象的创建时机

3. 依赖注入阶段

核心过程：

  • 解析 @Autowired、@Resource 等注解
  • 查找依赖对象并注入
  • 处理属性赋值（@Value）

关键点：

  • 注入方式：

    • 构造器注入（推荐）
    • Setter 方法注入
    • 字段注入

  • 扩展点：

最佳实践：

  • 使用构造器注入保证不可变依赖
  • 避免在 @PostConstruct 方法中使用依赖对象

4. 初始化阶段

核心过程：

  1. 调用 Aware 接口方法（BeanNameAware 等）
  2. 执行 BeanPostProcessor#postProcessBeforeInitialization
  3. 调用初始化方法：
    • @PostConstruct 注解方法
    • InitializingBean#afterPropertiesSet
    • 自定义 init-method
  4. 执行 BeanPostProcessor#postProcessAfterInitialization

关键扩展点：

典型应用：

  • 自定义注解处理
  • 代理对象的包装（如事务代理）

5. 使用阶段

核心特点：

  • Bean 已完全初始化
  • 可被其他 Bean 正常引用
  • 应用业务逻辑运行期

关键机制：

  • 单例 Bean 的缓存（DefaultSingletonBeanRegistry）
  • 作用域控制（@Scope）
  • AOP 代理的执行

监控方式：

6. 销毁阶段

销毁顺序：

  1. 执行 @PreDestroy 注解方法
  2. 调用 DisposableBean#destroy
  3. 执行自定义 destroy-method

关键点：

  • 触发时机：

    • 应用正常关闭（ConfigurableApplicationContext#close）
    • Web 应用上下文销毁
    • 非单例 Bean 的定期清理

  • 扩展点：

典型应用：

  • 数据库连接池的关闭
  • 临时文件的清理
  • 分布式锁的释放

理解这些阶段可以帮助你：

  1. 准确选择扩展点进行功能增强
  2. 诊断 Bean 创建过程中的问题
  3. 优化应用启动性能
  4. 合理管理资源生命周期
