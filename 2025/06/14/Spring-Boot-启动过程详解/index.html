Spring Boot 启动过程详解

Spring Boot 的启动过程是一个精心设计的自动化流程，下面我将详细解析其核心步骤和原理。

2.2 加载 ApplicationContextInitializer

通过 spring.factories 机制加载 META-INF/spring.factories 中配置的 ApplicationContextInitializer 实现类，用于在 Spring 容器刷新前进行初始化。

2.3 加载 ApplicationListener

同样通过 spring.factories 加载应用监听器，用于监听 Spring 应用事件。

2.4 推断主配置类

关键步骤详解

3.1 环境准备 (prepareEnvironment)

  1. 根据应用类型创建标准环境 (StandardServletEnvironment、StandardReactiveWebEnvironment 或 StandardEnvironment)
  2. 配置 PropertySources 和 Profiles
  3. 发布 ApplicationEnvironmentPreparedEvent 事件

3.2 创建应用上下文 (createApplicationContext)

根据应用类型创建不同的应用上下文：

  • Web 应用: AnnotationConfigServletWebServerApplicationContext
  • Reactive 应用: AnnotationConfigReactiveWebServerApplicationContext
  • 非 Web 应用: AnnotationConfigApplicationContext

3.3 准备上下文 (prepareContext)

  1. 将环境与上下文关联
  2. 后置处理上下文
  3. 应用所有 ApplicationContextInitializer
  4. 发布 ApplicationContextInitializedEvent 事件
  5. 注册主配置类的 Bean 定义
  6. 发布 ApplicationPreparedEvent 事件

3.4 刷新上下文 (refreshContext)

这是最核心的步骤，调用 AbstractApplicationContext.refresh() 方法：

  1. 准备刷新：初始化启动时间、活跃状态标志
  2. 获取内部 BeanFactory：obtainFreshBeanFactory()
  3. 准备 BeanFactory：配置标准 BeanFactory 特性（类加载器、EL 解析器等）
  4. 后置处理 BeanFactory：调用 postProcessBeanFactory() 方法
  5. 调用 BeanFactoryPostProcessor：执行 invokeBeanFactoryPostProcessors()
    • 这里会处理 @Configuration 类，包括 @ComponentScan、@Import 等
    • 特别重要的是 ConfigurationClassPostProcessor，它处理 @SpringBootApplication 注解
  6. 注册 BeanPostProcessor：registerBeanPostProcessors()
  7. 初始化消息源：initMessageSource()
  8. 初始化事件广播器：initApplicationEventMulticaster()
  9. 初始化特殊 Bean：onRefresh()
    • 对于 Web 应用，这里会创建内嵌的 Web 服务器（Tomcat/Jetty/Undertow）
  10. 注册监听器：registerListeners()
  11. 完成 BeanFactory 初始化：finishBeanFactoryInitialization()
    • 实例化所有非懒加载的单例 Bean
  12. 完成刷新：finishRefresh()
    • 发布 ContextRefreshedEvent 事件
    • 启动 Web 服务器

3.5 调用 Runner

最后，Spring Boot 会调用所有 ApplicationRunner 和 CommandLineRunner 的实现类，按照 @Order 注解或 Ordered 接口指定的顺序执行。

4. 自动配置原理

Spring Boot 自动配置的核心是 @SpringBootApplication 注解，它组合了三个关键注解：

其中 @EnableAutoConfiguration 是关键，它通过 @Import(AutoConfigurationImportSelector.class) 导入自动配置逻辑。

AutoConfigurationImportSelector 使用 SpringFactoriesLoader 加载 META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports 文件中定义的自动配置类。

每个自动配置类通常带有 @Conditional 系列注解，只有当特定条件满足时才会生效。

5. 内嵌服务器启动

对于 Web 应用，内嵌服务器的启动发生在 onRefresh() 阶段：

  1. ServletWebServerApplicationContext 会调用 createWebServer() 方法
  2. 通过 ServletWebServerFactory 创建服务器（默认是 Tomcat）
  3. 初始化 Servlet 上下文，注册 DispatcherServlet
  4. 启动 Web 服务器
