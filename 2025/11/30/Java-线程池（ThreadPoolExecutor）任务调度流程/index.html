详细执行流程（从 execute() 开始）

1️⃣ execute() 方法收到任务

流程开始。

2️⃣ 当前工作线程数 < corePoolSize？

核心线程未满 → 创建新核心线程直接执行任务：

特点：

  • 核心线程 永不回收（除非设置 allowCoreThreadTimeOut=true）
  • 任务立即执行，不进入队列

3️⃣ 核心线程已满 → 尝试放入阻塞队列

常见阻塞队列：

  • LinkedBlockingQueue（无界）
  • ArrayBlockingQueue（有界）
  • SynchronousQueue（不存储任务）

如果队列未满：

队列任务由核心线程逐个取出执行。

4️⃣ 队列已满 → 尝试创建非核心线程

非核心线程 = 核心线程之外，可回收

条件：

  • 当前线程数 < maximumPoolSize

如果成功，则新线程立即执行任务。

5️⃣ 队列满 + 线程已达最大 → 进入拒绝策略

触发：

默认策略：AbortPolicy（抛出 RejectedExecutionException）

还有三种：

策略 行为
AbortPolicy（默认） 直接抛异常
CallerRunsPolicy 调用方线程执行任务
DiscardPolicy 直接丢弃任务
DiscardOldestPolicy 丢弃队列最老任务，重新提交

🧠

最后总结成一段话

线程池调度流程就是：

核心线程优先 → 队列缓冲 → 非核心线程抗压 → 不够用时拒绝任务。

任务最终统一由 Worker 线程从队列中取出 run() 执行。

注意⚠️

无界队列（LinkedBlockingQueue 无容量上限）时，不会创建非核心线程

原因：ThreadPoolExecutor 的逻辑决定了——只要任务能成功入队，就绝不会去创建非核心线程。
